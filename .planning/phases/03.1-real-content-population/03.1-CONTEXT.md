## Implementation Decisions

### Blog migration
- **D-01:** Replace the 3 fabricated blog posts with the 3 real posts from blog.neporshiso.com:
  1. "The Moment I Realized The Importance Of Data Structures & Algorithms" (Dec 4, 2019) — `/data-structures-lesson/`
  2. "Small Group Projects - Tips for Success" (Nov 12, 2019) — `/small-group-projects/`
  3. "Welcome to the Blog!" (Oct 1, 2019) — `/welcome-to-the-blog/`
- **D-02:** Fetch the actual content from blog.neporshiso.com and convert to Markdoc `.mdoc` format with proper frontmatter (title, tags, excerpt, publishedDate, draft: false).
- **D-03:** Delete the 3 fabricated posts (building-a-personal-website-with-gatsby, learning-to-code-in-public, reflections-on-being-a-self-taught-developer).

### Podcast population
- **D-04:** User will provide the podcast list (<10 entries) during execution. The executor should prompt for the list before creating entries.
- **D-05:** For each podcast, use the iTunes Search API (`https://itunes.apple.com/search?term={name}&media=podcast&limit=1`) as a one-time lookup tool to download cover art. Save to `/public/assets/podcasts/{slug}.jpg` at 600x600 (artworkUrl600 from API response).
- **D-06:** Update each podcast `.mdoc`/`.yaml` entry with `coverImage: /assets/podcasts/{slug}.jpg` pointing to the local file.
- **D-07:** No build-time API calls — images are committed to the repo. Zero external dependencies at build.

### Book population
- **D-08:** User will provide the book list during execution with titles, authors, and reading status (reading/read/want-to-read).
- **D-09:** Cover images downloaded manually or from Open Library (`covers.openlibrary.org/b/isbn/{isbn}-M.jpg`) as a one-time operation. Save to `/public/assets/books/{slug}.jpg`.
- **D-10:** Portrait aspect ratio for book covers (natural book cover shape). The BookCard component already handles `aspect-[2/3]`.
- **D-11:** Add `isbn` field to book schema if using Open Library for covers (optional — can also download manually).

### Image sizing
- **D-12:** Podcast covers: 600x600px (iTunes artworkUrl600 default). PodcastCard displays at 64x64 — browser downscales.
- **D-13:** Book covers: portrait aspect, downloaded at natural resolution from source. BookCard displays at card width with `aspect-[2/3]` and `object-cover`.

### Claude's Discretion
- Markdoc formatting details for migrated blog posts (heading levels, code blocks, etc.)
- Whether to add `isbn` field to book schema or skip it (depends on cover image source)
- Podcast `.mdoc` vs `.yaml` format (current entries use `.mdoc`)
- Image compression/optimization before committing

</decisions>
