    1. Install Vercel adapter:

    2. Update `astro.config.mjs`:
       - Add import: `import vercel from '@astrojs/vercel';`
       - Add adapter: `adapter: vercel(),` inside defineConfig (after `site:`)
       - CRITICAL: Change keystatic from conditional to always-included. Replace:
         ```javascript
         ...(isProduction ? [] : [keystatic()]),
         ```
         With just:
         ```javascript
         keystatic(),
         ```
         Keystatic MUST be in the integrations array for BOTH dev and production (Anti-Pattern from research: "Including keystatic() conditionally"). The environment switch for storage mode happens in keystatic.config.ts, NOT here.
       - Remove the `const isProduction = process.env.NODE_ENV === 'production';` line (no longer needed in this file)
       - Keep all other integrations (react, markdoc, icon, sitemap) unchanged

       Final astro.config.mjs should be:
       ```javascript
       // @ts-check
       import { defineConfig } from 'astro/config';
       import tailwindcss from '@tailwindcss/vite';
       import react from '@astrojs/react';
       import markdoc from '@astrojs/markdoc';
       import icon from 'astro-icon';
       import keystatic from '@keystatic/astro';
       import sitemap from '@astrojs/sitemap';
       import vercel from '@astrojs/vercel';

       export default defineConfig({
         site: 'https://neporshiso.com',
         adapter: vercel(),
         vite: {
           plugins: [tailwindcss()],
         },
         integrations: [
           react(),
           markdoc(),
           icon(),
           sitemap(),
           keystatic(),
         ],
       });
       ```

    3. Update `keystatic.config.ts` — add environment-based storage switch (per D-05, D-06):
       - Add at top of file (before the config export):
         ```typescript
         const isProduction = process.env.NODE_ENV === 'production';
         ```
       - Change the storage field from:
         ```typescript
         storage: { kind: 'local' },
         ```
         To:
         ```typescript
         storage: isProduction
           ? {
               kind: 'github',
               repo: 'neporshiso/nep-personal-website',
             }
           : { kind: 'local' },
         ```
       - Do NOT change any collections or singletons definitions — they remain identical

    4. Create `src/pages/api/keystatic/[...params].ts`:
       - Create directory `src/pages/api/keystatic/` if it doesn't exist
       - File content:
         ```typescript
         import { makeHandler } from '@keystatic/astro/api';
         import keystaticConfig from '../../../../keystatic.config';
         import type { APIRoute } from 'astro';

         export const prerender = false;

         export const ALL: APIRoute = (context) => {
           return makeHandler({ config: keystaticConfig })(context);
         };
         ```
       - `export const prerender = false` is REQUIRED — this endpoint handles OAuth callbacks and must be server-rendered
       - `export const ALL` catches all HTTP methods (GET, POST, etc.)
       - The relative import `../../../../keystatic.config` traverses from `src/pages/api/keystatic/` to the project root

    5. Verify build works:
       ```bash
       npm run build
       ```
       The build should succeed. Vercel adapter handles the server routes. Sitemap should still generate for all static pages. The Keystatic API route will be server-rendered while all other pages remain static.
