    - If user said remove Syntax/ShopTalk Show, delete those `.mdoc` files
    - If user said keep them, leave them and add coverImage to their frontmatter after downloading art

    **Step 2: For each podcast, download cover art from iTunes Search API (per D-05, D-07, D-12).**
    For each podcast name:
    ```bash
    # Look up the podcast — use specific search terms to avoid wrong matches (RESEARCH.md Pitfall 3)
    SEARCH_TERM=$(python3 -c "import urllib.parse; print(urllib.parse.quote('PODCAST NAME HERE'))")
    RESULT=$(curl -s "https://itunes.apple.com/search?term=${SEARCH_TERM}&media=podcast&limit=1")

    # Verify correct podcast before downloading
    echo "$RESULT" | python3 -c "import sys,json; print(json.load(sys.stdin)['results'][0]['collectionName'])"

    # Download 600x600 cover art
    ARTWORK_URL=$(echo "$RESULT" | python3 -c "import sys,json; print(json.load(sys.stdin)['results'][0]['artworkUrl600'])")
    curl -sL "$ARTWORK_URL" -o "public/assets/podcasts/{slug}.jpg"
    ```

    **Step 3: Create `.mdoc` entry for each podcast (per D-06).**
    Follow the established pattern from existing entries. Each file at `src/content/podcasts/{slug}.mdoc`:
    ```yaml
    ---
    name: 'Podcast Name'
    link: 'https://podcast-website.com'
    coverImage: '/assets/podcasts/{slug}.jpg'
    category: 'Category'
    ---
    ```
    Body: empty (no body content — matches existing podcast entry pattern).

    Slug derivation: kebab-case from podcast name (e.g., "Planet Money" -> `planet-money.mdoc`).

    **CRITICAL per D-07:** All images must be committed to the repo. No build-time API calls. The iTunes API is only used during this one-time execution.

    **CRITICAL per RESEARCH.md anti-pattern:** Use `/assets/podcasts/{slug}.jpg` (public path), NOT `src/assets/images/podcasts/{slug}.jpg`.

    If iTunes returns wrong podcast (collectionName mismatch), try more specific search terms. If still wrong, note it and the user can provide an image manually.
  </action>

    **If ISBN provided:** Use Open Library Covers API:
    ```bash
    curl -sL "https://covers.openlibrary.org/b/isbn/{isbn}-L.jpg" \
      -o "public/assets/books/{slug}.jpg"

    # Verify cover was found (not 1x1 placeholder — RESEARCH.md Pitfall 4)
    SIZE=$(wc -c < "public/assets/books/{slug}.jpg")
    if [ "$SIZE" -lt 1000 ]; then
      echo "WARNING: Cover not found for {slug} — download manually from Amazon/Goodreads"
    fi
    ```

    **If ISBN not provided:** Search Open Library by title+author:
    ```bash
    # Look up ISBN first
    SEARCH=$(python3 -c "import urllib.parse; print(urllib.parse.quote('TITLE AUTHOR'))")
    curl -s "https://openlibrary.org/search.json?q=${SEARCH}&limit=1" | python3 -c "
    import sys,json
    d=json.load(sys.stdin)
    if d['docs']:
      isbns = d['docs'][0].get('isbn', [])
      print(isbns[0] if isbns else 'NO_ISBN')
    "
    ```
    Then use the ISBN with the covers API. If no ISBN found, note it for manual download.
