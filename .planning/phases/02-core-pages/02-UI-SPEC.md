## Spacing Scale

Declared values (multiples of 4 only). Tailwind v4 utility names map to these values via the default scale.

| Token | Value | Tailwind class | Usage |
|-------|-------|----------------|-------|
| xs | 4px | gap-1 / p-1 | Icon gaps, inline element padding |
| sm | 8px | gap-2 / p-2 | Compact element spacing, tag pill padding |
| md | 16px | gap-4 / p-4 | Default element spacing, card padding |
| lg | 24px | gap-6 / p-6 | Section-internal spacing |
| xl | 32px | gap-8 / p-8 | Layout gaps between distinct blocks |
| 2xl | 48px | gap-12 / py-12 | Major section breaks (e.g. page intro from grid) |
| 3xl | 64px | gap-16 / py-16 | Page-level top/bottom padding (reserved for hero areas) |

Exceptions:
- Touch targets (ThemeToggle, mobile hamburger): minimum 44px × 44px — already established in Phase 1, carried forward. Use `min-w-[44px] min-h-[44px]`.
- Content column max-width: `max-w-3xl mx-auto px-4` — established in BaseLayout, must not be overridden per page.

## Typography

All sizes use the system font stack from global.css. No custom typeface loaded.

| Role | Size | Tailwind class | Weight | Weight class | Line Height | Usage |
|------|------|----------------|--------|--------------|-------------|-------|
| Body | 16px | text-base | Regular (400) | font-normal | 1.6 | Body copy, bio text, descriptions |
| Label | 14px | text-sm | Regular (400) | font-normal | 1.5 | Nav links, captions, tag pills, metadata (year, status, author) |
| Heading | 20px | text-xl | Semibold (600) | font-semibold | 1.3 | Card titles, section subheadings (h2 within content area) |
| Display | 30px | text-3xl | Bold (700) | font-bold | 1.2 | Page-level h1 (e.g. "About", "Portfolio") |

Notes:
- Two weights in active use: regular (400) for body/label, semibold (600) for headings — bold (700) reserved for display h1 only.
- `line-height: 1.6` on body is already set globally in `global.css` — do not override.
- Page h1 pattern `text-3xl font-bold mb-6` is established in `about.astro` — apply consistently across all pages.
- Code blocks: `ui-monospace` stack, no size override (inherits body 16px), used in blog phase — referenced here for completeness.

Source: global.css line 46 (line-height 1.6), about.astro line 11 (text-3xl font-bold mb-6), Nav.astro line 17 (font-semibold text-lg for site name), Nav.astro line 27 (text-sm for nav links).

## Color

All values taken directly from global.css. No new colors introduced in Phase 2.

| Role | Light mode | Dark mode | Tailwind / CSS var | Usage |
|------|-----------|-----------|-------------------|-------|
| Dominant (60%) | #fdf6e3 | #1d2021 | `bg-[var(--bg)]` / `var(--bg)` | Page background, all surface backgrounds |
| Secondary (30%) | #d5c4a1 border / #fdf6e3 card bg | #504945 border / #1d2021 card bg | `border-[var(--border)]` | Nav border, footer border, card borders, dividers |
| Text primary | #3c3836 | #ebdbb2 | `text-[var(--text)]` | All body text, headings, labels |
| Text muted | #7c6f64 | #7c6f64 | `text-[var(--muted,#7c6f64)]` | Metadata, captions, footer copyright, secondary labels (year, author, status badges) |
| Accent (10%) | #b57614 (large/decorative only) / #d79921 (interactive text, dark) | #d79921 | `text-[var(--accent)]` | Exact reserved list below |
| Accent hover | #af3a03 | #af3a03 | `text-[var(--accent-hover)]` | Hover state on accent-colored interactive elements |
| Destructive | not applicable | not applicable | n/a | No destructive actions in Phase 2 |

**Accent reserved for (explicit list — no other elements use accent color):**
1. External links within body copy (project detail write-ups, bio text)
2. "Get in touch" contact CTA — the single primary call-to-action
3. Active/current nav link indicator (underline or color, not background fill)
4. Reading status badge "reading" on Books page (to signal in-progress, not for decoration)

**WCAG AA compliance carried forward from Phase 1 (global.css lines 1–16):**
- `#b57614` on `#fdf6e3`: 3.50:1 — use only for large text (heading-size) or decorative in light mode.
- `#d79921` on `#fdf6e3`: passes AA for normal text — use this token for interactive links/text in light mode.
- `#d79921` on `#1d2021`: 5.13:1 — passes AA in dark mode.
- Implementation note: The CSS var `--color-accent` is set to `#b57614`. For interactive inline text links, apply `#d79921` explicitly via `text-[#d79921]` in light mode or use a `light:` variant when the Iconify/accent pattern needs AA compliance on normal-size text.

## Interaction Patterns

| Pattern | Implementation |
|---------|---------------|
| Hover feedback | `hover:opacity-70` on nav links (established in Nav.astro); `hover:opacity-80` on cards |
| Focus visible | Browser default focus ring — do not suppress `outline`. All interactive elements must show `:focus-visible` ring |
| Touch targets | All tap targets minimum 44×44px (established in Phase 1 for toggle/hamburger) |
| Active nav link | Current page link: `aria-current="page"` attribute + accent color text |
| Image loading | `loading="lazy"` on all card images (below the fold). `loading="eager"` on project detail hero only |
| Empty states | Text only — no illustrations. See Copywriting Contract below |
| Error states | Text only — inline message in content area. See Copywriting Contract below |
| External links | `target="_blank" rel="noopener noreferrer"` on all outbound hrefs |
